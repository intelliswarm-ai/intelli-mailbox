## 5. Security Considerations and Acknowledged Risks

### 5.1 Architectural Overview

The Software accesses Your Email Account by attaching to a real Chrome
(or compatible Chromium) browser via the Chrome DevTools Protocol
("CDP"). The browser is launched with a dedicated user profile under
Your user account and the Software communicates with that browser over
a local-only port (default `localhost:9222`) that the browser itself
exposes for control purposes. The Software does **not** use the Gmail
API, an OAuth refresh token, or Your Google password.

This architecture has specific properties You should understand and
accept before installing the Software.

### 5.2 Local Machine Trust Boundary

Any program running on Your computer under the same user account as
the Software can in principle interact with the same browser the
Software does. This is true of any browser-automation tool. **The
Software's security boundary is therefore the trust boundary of Your
local user account.** You are responsible for maintaining that
boundary by, at minimum:

  - keeping Your operating system patched and Your antivirus active,
  - not running untrusted programs under the same user account that
    runs the Software,
  - not granting Remote Desktop, screen-sharing, or remote-access
    rights to untrusted parties while the Software is running, and
  - protecting the dedicated Chrome profile directory at
    `%LOCALAPPDATA%\.intelliswarm\intellimailbox-chrome` (or its
    macOS / Linux equivalent) from disk-level disclosure (backup
    services, OS snapshots, file-sync tools).

You acknowledge and accept that, by the nature of Your use of the

  - **CDP-port exposure.** Any other process running under Your user

    (firewalls scoped to interface, per-process sandboxes) are
    *Your* responsibility, not ours.
  - **Profile-directory exposure.** The dedicated Chrome profile
    directory contains session cookies and cached browser state
    that, if copied off the machine, may grant the holder access to
    the Email Account without further authentication. Backup,
    sync, screenshot, and remote-management tools that read that
    directory may therefore expose the Email Account.
  - **Supply-chain compromise.** A malicious update to Chrome,
    Chromium, Ollama, Playwright, swarmai-core, the bundled JDK, or

    the confidentiality of email data it reads.
  - **Browser-side vulnerability.** A vulnerability in Chrome
    itself, or a malicious browser extension installed in the
    dedicated Chrome profile, may expose the Email Account or
    permit lateral access to other browser data on the machine.
  - **Network misconfiguration.** While the CDP control port and
    the Software's own web UI bind to `localhost` only, a
    misconfigured network proxy, VPN, anti-virus middleware, or
    third-party reverse-proxy on Your machine may unintentionally
    expose those ports to remote parties.
  - **Defects in the Software.** Programming errors in the Software
    or its update mechanism may create or amplify the above risks
